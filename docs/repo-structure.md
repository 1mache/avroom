# Repository Structure

Annotated tour of every top-level entry. Generated artifacts (`__pycache__/`, `node_modules/`, `*.egg-info/`, `outputs/`, etc.) are mentioned briefly but not enumerated.

```
C:/Avroom/avroom/
├── .cursor/              - Cursor editor config (skills, hooks, plans)
│   ├── skills/           - SKILL.md files invocable from chat
│   └── hooks/            - shell scripts triggered by Cursor events
├── .git/                 - git metadata
├── .gitignore
├── avroom_context.md     - legacy project notes (predates docs/)
├── docs/                 - this docs tree (you are here)
├── fastApi-app/          - FastAPI backend service
├── react-front/          - React + Vite SPA
├── TestModules/          - the avroom_object_removal Python package
├── TrellisModule/        - the avroom_trellis image-to-3D wrapper package
└── requirements.txt      - root requirements; installs local packages editable
```

## `TestModules/`

The Python package. Sources live in `src/` but are imported as `avroom_object_removal.*` thanks to `[tool.setuptools.package-dir]` in [`TestModules/pyproject.toml`](../TestModules/pyproject.toml).

```
TestModules/
├── pyproject.toml      - declares avroom-object-removal package
├── src/                - imported as avroom_object_removal
│   ├── __init__.py     - re-exports ObjectRemover
│   ├── core/
│   │   ├── interfaces.py    - IDepthFacade, IImageAdapter, IInpainter, ISegmentationRoutingStrategy
│   │   └── objectRemover.py - the orchestrator (the only public class)
│   ├── ai_engines/
│   │   ├── depth/           - OptimizedDepthFacade, ImageDepthMapper
│   │   ├── segmentation/    - SamFacadeSingleton, SamImageAdapter
│   │   ├── inpainting/      - LamaInpainter, StableDiffusionInpainter, HybridInpainter
│   │   └── 3dRreconstruction/Hunyuan3D-2.1/  - upstream checkout, NOT wired in
│   ├── routing/             - segmentation routing strategies
│   ├── utils/               - MaskRefiner, MaskOverlapRGBAComposer, DebugImageSaver, ImageAdapterFactory
│   └── GuiTestClicker.py    - manual GUI test harness (not part of API path)
├── tests/              - integration scripts (not pytest)
├── inputs/             - test images
├── outputs/            - debug PNGs written every run (ignored)
├── checkpoints/        - SAM weights here by default
├── depthMaps/          - cached depth model outputs
└── venv/               - local venv (ignored)
```

See [ai-pipeline/README.md](ai-pipeline/README.md) for details.

## `TrellisModule/`

Standalone image-to-3D wrapper package (`avroom_trellis`). It calls the public Hugging Face Space `microsoft/TRELLIS.2` via `gradio_client` and returns a GLB.

```
TrellisModule/
├── pyproject.toml      - declares avroom-trellis package + deps
├── src/                - imported as avroom_trellis
│   ├── __init__.py     - exports Trellis3DGenerator + Quality
│   ├── generator.py    - Trellis3DGenerator.generate(...)
│   ├── quality.py      - FAST/BALANCED/HIGH presets
│   ├── image_input.py  - input normalization (bytes/ndarray/PIL/path -> PIL RGBA)
│   └── output_writer.py
└── tests/
    └── test_smoke.py   - manual ObjectRemover -> Trellis smoke test
```

This module is **not wired into** `fastApi-app/` today. See [trellis-module.md](trellis-module.md).

## What is not architectural

- `*.egg-info/` directories — distutils/setuptools metadata, regenerated by `pip install`.
- `__pycache__/`, `*.pyc` — Python bytecode.
- `node_modules/` — npm install output.
- `TestModules/outputs/`, `fastApi-app/images/tmp/`, `TestModules/depthMaps/` — runtime artifacts.
- `TestModules/src/ai_engines/3dRreconstruction/Hunyuan3D-2.1/.git/` — nested upstream repo metadata only.

Anything in those locations is downstream of the code in `src/`, not part of the architecture.
