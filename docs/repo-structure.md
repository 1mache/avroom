# Repository Structure

Annotated tour of every top-level entry. Generated artifacts (`__pycache__/`, `node_modules/`, `*.egg-info/`, `outputs/`, etc.) are mentioned briefly but not enumerated.

```
C:/Avroom/avroom/
├── .cursor/              - Cursor editor config (skills, hooks, plans)
│   ├── skills/           - SKILL.md files invocable from chat
│   └── hooks/            - shell scripts triggered by Cursor events
├── .git/                 - git metadata
├── .gitignore
├── avroom_context.md     - legacy project notes (predates docs/)
├── docs/                 - this docs tree (you are here)
├── fastApi-app/          - FastAPI backend service

├── TestModules/          - the avroom_object_removal Python package (depth/segmentation/inpainting/3D)

```

## `TestModules/`

The Python package. Sources live in `src/` but are imported as `avroom_object_removal.*` thanks to `[tool.setuptools.package-dir]` in [`TestModules/pyproject.toml`](../TestModules/pyproject.toml). Internally the package follows a Facade + Strategy split per AI domain (depth, segmentation, inpainting, 3D reconstruction) plus a routing strategy that bridges depth → segmentation.

```
TestModules/
├── pyproject.toml      - declares avroom-object-removal package
├── src/                - imported as avroom_object_removal
│   ├── __init__.py                       - re-exports ObjectRemover, the four Facades, ABCs, and concrete strategies
│   ├── core/
│   │   ├── __init__.py                   - exports ObjectRemover
│   │   └── object_remover.py             - master Facade orchestrating the pipeline
│   ├── ai_engines/
│   │   ├── depth/                        - DepthMappingFacade + DepthMappingStrategy (ABC)
│   │   │   └── strategies/               - DepthAnythingMappingStrategy, NearFarBlendedDepthMappingStrategy
│   │   ├── segmentation/                 - ImageSegmentationFacade + ImageSegmentationStrategy (ABC) + SamImageAdapter
│   │   │   └── strategies/               - SamSegmentationStrategy
│   │   ├── inpainting/                   - ImageInpaintingFacade + ImageInpaintingStrategy (ABC)
│   │   │   └── strategies/               - LamaInpaintingStrategy, StableDiffusionInpaintingStrategy, HybridInpaintingStrategy
│   │   └── reconstruction_3d/            - Reconstruction3DFacade + ABC + quality presets + GLB writer
│   │       ├── _backends/openlrm_v10/   - vendored OpenLRM v1.0 inference (`lrm/`)
│   │       └── strategies/               - TriposrReconstructionStrategy (default), plus OpenLRM/Trellis/etc. (optional)
│   ├── routing/                          - SegmentationRoutingStrategy (ABC)
│   │   └── strategies/                   - BoundaryVarianceRoutingStrategy, CenterOfMassRoutingStrategy
│   └── utils/                            - DebugImageSaver, MaskRefiner, BgraCutoutComposer
├── tests/              - integration / smoke scripts (not pytest)
├── inputs/             - test images
├── outputs/            - debug PNGs written every run (ignored)
├── checkpoints/        - SAM weights here by default
├── depthMaps/          - cached depth model outputs
└── venv/               - local venv (ignored)
```

See [ai-pipeline/README.md](ai-pipeline/README.md) for details.

## What is not architectural

- `*.egg-info/` directories — distutils/setuptools metadata, regenerated by `pip install`.
- `__pycache__/`, `*.pyc` — Python bytecode.
- `node_modules/` — npm install output.
- `TestModules/outputs/`, `fastApi-app/tmp/images/`, `TestModules/depthMaps/` — runtime artifacts.

Anything in those locations is downstream of the code in `src/`, not part of the architecture.
